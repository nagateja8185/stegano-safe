How Decryption Works

LSB Extraction

  • The tool reads the least significant bits from each pixel channel
  • The hidden encrypted payload is reconstructed from the bits
  • CRC32 checksum verifies data integrity

SHA-256 Verification

  • The encrypted data hash is computed and verified
  • Any tampering will cause hash mismatch and rejection

AES-256-GCM Decryption

  • Your password re-derives the AES-256 key via PBKDF2
  • GCM tag authentication ensures password correctness
  • Wrong password → authentication failure (not garbage data)
Decrypted files are sent directly to your browser. They are never stored on the server in unencrypted form.
